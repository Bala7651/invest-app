Add `uniqueIndex('price_alerts_symbol_unique').on(t.symbol)`.

2. **Migration** — Run `cd invest-app && npx drizzle-kit generate` to produce the new SQL migration file in `drizzle/`. This updates `drizzle/migrations.js` automatically. Verify the generated SQL creates the `price_alerts` table with correct columns.

3. **alertService.ts** — Create `invest-app/src/features/alerts/services/alertService.ts`. Follow the watchlistService pattern (import db from `../../../db/client`, price_alerts from `../../../db/schema`, eq from `drizzle-orm`). Export:
   - `AlertRow` interface: `{ id, symbol, name, upper_price: number|null, lower_price: number|null, upper_status: 'active'|'triggered', lower_status: 'active'|'triggered' }`
   - `upsertAlert(symbol, name, upper_price, lower_price)` — delete-then-insert pattern (same as daily_summaries). Reset both statuses to 'active' on upsert.
   - `getAll()` — select all rows
   - `deleteAlert(id)` — delete by id
   - `markTriggered(symbol, direction: 'upper'|'lower')` — update the relevant status column to 'triggered'
   - `reEnableDirection(symbol, direction)` — update the relevant status column back to 'active'
   - `getActiveSymbols()` — returns unique symbol list where at least one direction is 'active'

4. **alertMonitor.ts** — Create `invest-app/src/features/alerts/services/alertMonitor.ts`. Export `checkAlerts(quotes: Record<string, QuoteLike>)` where `QuoteLike = { symbol: string, name: string, price: number | null }`. Logic: get all alerts from alertService, iterate each, check upper_price (active + price >= target) and lower_price (active + price <= target). On match: call `markTriggered`, then call `fireAlertNotification`. The `fireAlertNotification` function uses `expo-notifications scheduleNotificationAsync` with `trigger: { type: SchedulableTriggerInputTypes.TIME_INTERVAL, seconds: 1, channelId: 'price-alerts' }` — NOT `trigger: null` (unreliable from background task per GitHub issue #21267). Notification content: title `${name} price alert`, body `${name} crossed ${direction} ${targetPrice} - current: ${currentPrice.toFixed(2)}`.

Test: upsertAlert calls delete then insert, getAll returns rows, deleteAlert calls delete with correct id, markTriggered updates correct column. Create `alertMonitor.test.ts` mocking alertService and expo-notifications. Test: checkAlerts fires notification when price >= upper_price (active), fires when price <= lower_price (active), does NOT fire when status is 'triggered', does NOT fire when price is null, calls markTriggered before fireAlertNotification.

Install expo-notifications: `cd invest-app && npx expo install expo-notifications expo-background-task expo-task-manager expo-intent-launcher`

Import `TaskManager` from `expo-task-manager`, `BackgroundTask` from `expo-background-task`, `isMarketOpen` from `../../market/marketHours`, `getAll` from `./alertService`, `getQuotes` from `../../../services/stockService`, `checkAlerts` from `./alertMonitor`.

   Export `ALERT_CHECK_TASK = 'PRICE_ALERT_CHECK'`.

   `defineTask(ALERT_CHECK_TASK, async () => { ... })`:
   - If `!isMarketOpen()` return `BackgroundTaskResult.Success` (skip outside trading hours)
   - Get all alerts, filter to those with at least one 'active' direction
   - If no active alerts, return Success
   - Extract unique symbols, call `getQuotes(symbols)` directly (NOT from quoteStore — background task runs in separate JS context)
   - Build quotes map, call `checkAlerts(quotesMap)`
   - Return Success. Catch errors, return Failed.

   Export `registerAlertTask()`: check `BackgroundTask.getStatusAsync()`, if Available then `registerTaskAsync(ALERT_CHECK_TASK, { minimumInterval: 15 })`.

   Export `unregisterAlertTask()`: call `BackgroundTask.unregisterTaskAsync(ALERT_CHECK_TASK)`.

2. **alertStore.ts** — Create `invest-app/src/features/alerts/store/alertStore.ts`. Zustand store following watchlistStore pattern. State: `alerts: AlertRow[]`. Actions:
   - `loadFromDb()` — calls `alertService.getAll()`, sets alerts
   - `upsertAlert(symbol, name, upper, lower)` — calls `alertService.upsertAlert`, updates state (filter out old row for symbol, add new). After upsert: if any active alerts exist, call `registerAlertTask()`. Also update persistent monitoring notification count.
   - `deleteAlert(id)` — calls `alertService.deleteAlert`, filters state. If no alerts remain, call `unregisterAlertTask()` and dismiss monitoring notification.
   - `reEnable(symbol, direction)` — calls `alertService.reEnableDirection`, then `loadFromDb()` to refresh. Re-register background task if needed.
   - `getBySymbol(symbol)` — find alert row by symbol
   - `activeCount()` — count alerts where upper_status=active OR lower_status=active

   Import and use `updateMonitoringNotification(count)` and `dismissMonitoringNotification()` helpers. Define these in alertStore.ts (or inline): use `Notifications.scheduleNotificationAsync` with identifier `'monitoring-status-persistent'`, sticky: true, channelId: `'monitoring-status'`, body: `Monitoring ${count} alerts`. For dismiss: `Notifications.dismissNotificationAsync('monitoring-status-persistent')`. Use `trigger: null` for monitoring notification (foreground-only, safe).
