  <files>invest-app/src/features/summary/types.ts, invest-app/src/features/summary/services/summaryService.ts</files>
  <action>
1. Create `src/features/summary/types.ts`:
   - `SummaryEntry`: `{ id: number; symbol: string; date: string; content: string; created_at: Date }` — matches daily_summaries schema row
   - `Credentials`: `{ apiKey: string; modelName: string; baseUrl: string }`
   - `ERROR_PREFIX = '__ERROR__:'` constant — used to mark failed stocks in content field

2. Create `src/features/summary/services/summaryService.ts`:

   **Date helpers (Taipei timezone):**
   - `getTodayISO()`: Returns `YYYY-MM-DD` using `toLocaleString('en-US', { timeZone: 'Asia/Taipei' })` then manual format. Never use `.toISOString()` (UTC).
   - `getCutoffISO()`: Returns ISO date 14 days before today in Taipei time.
   - `isCatchUpNeeded()`: Returns true if: (1) Taipei time >= 12:30, (2) weekday (not Sat/Sun), (3) not a holiday (use `isHoliday` from marketHours.ts — pass a Taipei-converted Date). Export for _layout.tsx.

   **TWSE Index fetch:**
   - `fetchTWIX()`: Fetch `https://openapi.twse.com.tw/v1/exchangeReport/MI_INDEX` with 10s AbortSignal timeout. Find entry where `指數 === '發行量加權股價指數'`. Return `{ close: number, change: number, changePct: number }` or null. Handle `漲跌` sign (`+` vs `-`) for change direction.

   **AI summary call:**
   - `SUMMARY_SYSTEM_PROMPT`: System prompt requesting 2-3 sentence Traditional Chinese plain text (no JSON, no markdown). See RESEARCH.md for exact prompt.
   - `buildSummaryPrompt(symbol, name, quote)`: Builds user prompt with real price data. Uses QuoteData-like shape but only needs price, change, changePct, prevClose.
   - `buildIndexSummaryPrompt(indexData)`: Builds prompt for TAIEX index using close/change/changePct.
   - `callSummaryMiniMax(systemPrompt, userPrompt, credentials)`: Same fetch pattern as minimaxApi.ts callMiniMax but returns raw `string` content (not parsed JSON). Uses `credentials.baseUrl + '/chat/completions'`, same headers/body shape. 30s timeout. `max_tokens: 300` (shorter than analysis). `temperature: 0.3`.

   **SQLite CRUD:**
   - `upsertSummary(symbol, date, content)`: Delete existing row where `symbol === symbol AND date === date`, then insert new row. Uses drizzle `db.delete()` + `db.insert()` on `daily_summaries`.
   - `purgeOldSummaries()`: Delete rows where `daily_summaries.date < getCutoffISO()` using drizzle `lt()`. ISO text comparison is lexicographically correct.
   - `loadAllSummaries()`: Select all from daily_summaries ordered by `desc(date), desc(created_at)`. Return `SummaryEntry[]`.
   - `hasSummaryForDate(date: string)`: Select count where `daily_summaries.date === date` limit 1. Return boolean. Used by catch-up check.

   **Imports:** `db` from `../../../db/client`, `daily_summaries` from `../../../db/schema`, `eq, lt, and, desc` from `drizzle-orm`, `isHoliday` from `../../market/marketHours`.
