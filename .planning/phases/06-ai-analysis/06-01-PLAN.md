    - fetchAnalysis re-fetches when cache exists but age >= 30 minutes
    - fetchAnalysis sets loading[symbol]=true during fetch, false after
    - fetchAnalysis stores error message in errors[symbol] on failure, loading=false
    - fetchAnalysis does NOT throw — errors captured in store
    - clearAnalysis clears all cache, cachedAt, loading, errors
  </behavior>
  <action>
    Create `analysisStore.ts` as Zustand store modelled on chartStore pattern:

    ```typescript
    interface AnalysisState {
      cache: Record<string, AnalysisResult>;
      cachedAt: Record<string, number>;
      loading: Record<string, boolean>;
      errors: Record<string, string | null>;
      fetchAnalysis: (symbol: string, quote: QuoteData, credentials: Credentials) => Promise<void>;
      clearAnalysis: () => void;
    }
    ```

    TTL check: `const cachedAt = get().cachedAt[symbol] ?? 0; if (get().cache[symbol] && Date.now() - cachedAt < TTL_MS) return;` where TTL_MS = 30 * 60 * 1000.

    On success: set cache[symbol], cachedAt[symbol]=Date.now(), loading[symbol]=false, errors[symbol]=null.
    On failure: set loading[symbol]=false, errors[symbol]=String(e). Do NOT rethrow.

    - Test TTL expired: manipulate cachedAt to be old, second call re-fetches
    - Test error: mock callMiniMax to throw, verify errors[symbol] set, loading false
