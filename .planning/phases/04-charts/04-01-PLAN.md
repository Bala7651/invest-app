2. Create `src/features/charts/services/historicalService.ts`:
   - Create a SEPARATE `RequestQueue` instance (copy the class from stockService.ts pattern, do NOT import the private queue) with 3-second spacing for FinMind/TWSE historical endpoints.
   - Export `twseDateToTimestamp(twseDate: string): number` — splits on `/`, adds 1911 to ROC year, returns `new Date(year, month-1, day).getTime()`.
   - Internal `fetchFinMindDaily(stockId, startDate)` — calls `https://api.finmindtrade.com/api/v4/data` with `dataset=TaiwanStockPrice`, `data_id=stockId`, `start_date=YYYY-MM-DD`. Maps response: `row.date` -> timestamp, `row.open`, `row.max` (high), `row.min` (low), `row.close`, `row.Trading_Volume` (volume). Use AbortSignal.timeout(10_000).
   - Internal `fetchTWSEMonthly(stockId, year, month)` — calls `https://www.twse.com.tw/exchangeReport/STOCK_DAY?response=json&date=${YYYYMM01}&stockNo=${stockId}`. Parse ROC dates via `twseDateToTimestamp`. Parse comma-separated numeric strings (e.g., "1,234"). Fields: `[date, volume, value, open, high, low, close, change, transactions]`.
   - Internal `getDateRange(timeframe: Timeframe): string` — returns start date as YYYY-MM-DD string. 1D: 1 day ago, 5D: 7 calendar days ago (covers 5 trading days), 1M: 1 month ago, 6M: 6 months ago, 1Y: 1 year ago.
   - Export `fetchCandles(symbol: string, timeframe: Timeframe): Promise<OHLCVPoint[]>`:
     - For 1M/6M/1Y: try FinMind first. If FinMind returns empty or errors, fall back to TWSE STOCK_DAY (fetch each required month sequentially via queue).
     - For 1D/5D: use TWSE STOCK_DAY daily candles (fetch current month + previous month if needed for 5D). Note: intraday 5-min/30-min candles deferred since Fugle requires API key registration. Daily candles for 1D show one day's single candle or last few trading days with daily granularity. This is the simplest v1 approach; Fugle intraday can be added in a future iteration when user sets up the API key.
     - Always sort result ascending by timestamp before returning.
     - All fetch calls go through the historicalService's own RequestQueue.

   - Follow Zustand v5 pattern from quoteStore.ts.
   - State shape:
     ```typescript
     interface ChartState {
       cache: Record<string, OHLCVPoint[]>; // key: `${symbol}:${timeframe}`
       loading: Record<string, boolean>;
       errors: Record<string, string | null>;
       fetchCandles: (symbol: string, timeframe: Timeframe) => Promise<void>;
       getCandles: (symbol: string, timeframe: Timeframe) => OHLCVPoint[] | undefined;
       clearCache: (symbol?: string) => void;
     }
     ```
   - `fetchCandles`: checks cache first — if hit, returns without fetching. Sets loading[key]=true, calls historicalService.fetchCandles, stores result in cache, sets loading[key]=false. On error, sets errors[key] with error message.
   - `getCandles`: returns `cache[${symbol}:${timeframe}]` or undefined.
   - `clearCache`: if symbol provided, clears all keys starting with `${symbol}:`; if no symbol, clears entire cache.
   - Export `useChartStore`.
