  <action>
    1. Run `npx create-expo-app@latest --template default@sdk-55 invest-app` in `/Users/linmini/invest/`.

    2. Install dependencies (in invest-app/):
       ```bash
       npx expo install expo-router react-native-safe-area-context react-native-screens expo-linking expo-constants expo-status-bar
       npm install nativewind react-native-reanimated
       npm install --save-dev tailwindcss@^3.4.17
       npx expo install react-native-pager-view
       npm install zustand
       ```

    3. Create `babel.config.js` — compose NativeWind + inline-import (for drizzle in Plan 02):
       ```javascript
       module.exports = function (api) {
         api.cache(true);
         return {
           presets: [
             ['babel-preset-expo', { jsxImportSource: 'nativewind' }],
             'nativewind/babel',
           ],
           plugins: [
             ['inline-import', { extensions: ['.sql'] }],
           ],
         };
       };
       ```
       Note: Install `npm install --save-dev babel-plugin-inline-import` for the inline-import plugin.

    4. Create `metro.config.js` — compose NativeWind + .sql extension:
       ```javascript
       const { getDefaultConfig } = require('expo/metro-config');
       const { withNativeWind } = require('nativewind/metro');
       const config = getDefaultConfig(__dirname);
       config.resolver.sourceExts.push('sql');
       module.exports = withNativeWind(config, { input: './global.css' });
       ```

    5. Create `tailwind.config.js` with cyberpunk theme tokens (exact hex values from CONTEXT.md):
       - bg: #050508, surface: #0D0D14, border: #1A1A2E
       - primary: #4D7CFF, secondary: #8B5CF6
       - stock-up: #00E676, stock-down: #FF1744
       - text: #E0E0E0, muted: #6B7280
       Include `presets: [require('nativewind/preset')]` and content paths `./app/**/*.{ts,tsx}` and `./src/**/*.{ts,tsx}`.

    6. Create `global.css` with `@tailwind base; @tailwind components; @tailwind utilities;`

    7. Create `nativewind-env.d.ts` at project root with `/// <reference types="nativewind/types" />`

    8. Update `tsconfig.json`:
       - extends: "expo/tsconfig.base"
       - compilerOptions.strict: true
       - compilerOptions.paths: { "@/*": ["./src/*"] }

    9. Create `src/theme/tokens.ts` exporting all color hex values as a `colors` object (same values as tailwind.config.js, for programmatic use outside className).

    10. Update `.gitignore` to include: `.env`, `.env.*`, `*.keystore`, `google-services.json`, `GoogleService-Info.plist`, `credentials.json`, `eas-service-account.json`, `/android/`, `/ios/`, `/dist/`, `/.expo/`, `/node_modules/`.

    CRITICAL: Use tailwindcss@^3.4.17, NOT tailwindcss@4. NativeWind v4 only supports Tailwind CSS v3.
    CRITICAL: `nativewind/babel` goes in presets array, NOT plugins array.
    CRITICAL: After all config changes, test with `npx expo start --clear` to bust Metro cache.
